## Overview

Voting campaigns allow server admins to run community votes where members vote for their favorites, and winners automatically receive rewards from a prize pool.

## How Voting Works

### Campaign Creation

Admins create voting campaigns with:

- **Campaign Name**: Title of the vote
- **Description**: What members are voting on
- **Options**: Voting choices (candidates, items, etc.)
- **Prize Pool**: Total rewards to distribute
- **Voting Period**: Start and end dates
- **Vote Limits**: How many votes each member gets

### Member Participation

Members can:

- **View Campaigns**: See active voting campaigns
- **Cast Votes**: Vote for their favorites
- **View Results**: See current standings
- **Receive Rewards**: Get rewards if their choice wins

## Campaign Types

### Single Choice

Members vote for one option:

- **Best of X**: Choose the best option
- **Election**: Vote for a candidate
- **Preference**: Express preference

### Multiple Choice

Members can vote for multiple:

- **Top 3**: Select top 3 choices
- **Ranked Choice**: Rank options
- **Multiple Selections**: Select multiple favorites

## Prize Distribution

### Winner Rewards

- **Top Winners**: Rewards for top vote recipients
- **Prize Pool**: Total rewards available
- **Distribution**: How prizes are distributed
- **Automatic**: Rewards given automatically

### Participation Rewards

Some campaigns offer:

- **Voter Rewards**: XP for all voters
- **Participation Badges**: Badges for voting
- **Random Prizes**: Random rewards for voters

## Campaign Features

### Real-Time Results

- **Live Updates**: See results update in real-time
- **Leaderboard**: Current standings
- **Vote Counts**: How many votes each option has
- **Trends**: Voting trends over time

### Campaign Analytics

Admins can view:

- **Total Votes**: Number of votes cast
- **Participation Rate**: Percentage of members voting
- **Vote Distribution**: How votes are spread
- **Trend Analysis**: Voting patterns
